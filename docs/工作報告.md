# AI 助教 API 故障診斷與多核心備援優化工作報告

本報告總結了針對「AI 機器學習助教」聊天機器人 API 的故障排查、解決方案設計、防禦性開發，以及最終多核心備援與自動降級機制的實作成果。

## 1. 核心問題排查與診斷

在開發與測試階段，AI 助教面臨了幾項嚴峻的 API 錯誤：
1. **OpenAI API 429 Quota Exceeded**：原帳號的免費 API 額度過期或用盡，導致服務不可用。
2. **Gemini API 429 Rate Limits**：受限於 Google AI Studio 免費版每分鐘的呼叫頻率限制（RPM），或受部署伺服器（Render）IP 地區限流。
3. **OpenRouter API 401 Unauthorized**：部署平台的環境變數在設定時常因多餘引號（如 `"sk-or-v1-..."`）或空格，造成金鑰解析錯誤與 Authentication header 遺失。
4. **OpenRouter API 404 Model Unavailable**：指定之 `meta-llama/llama-3-8b-instruct:free` 免費模型突然被下架或改為收費版。

---

## 2. 解決方案與優化設計

為確保 Demo 與部署階段 100% 不中斷，本專案重構了後端核心對話架構，設計了以下四層保障：

### A. 防禦性金鑰淨化機制 (Sanitization)
針對部署環境變數解析問題，實作了防禦性清洗函數：
* 自動清除金鑰字串的前後空白。
* 自動過濾不慎填入的單引號 (`'`) 與雙引號 (`"`")。
* 新增啟動遮罩日誌（如 `sk-or-v1...abcd (length: 53)`），方便在 Render Log 中快速核對金鑰狀態。

### B. 動態免費路由模型 (OpenRouter Free Router)
將 OpenRouter 指定模型改為 **`openrouter/free`**。此 ID 會由 OpenRouter 官方動態分配當下可用的免費開源大模型（如 Llama 3、Qwen、Gemma 等），避免模型下架引起的 404 崩潰。
* ⚠️ **特別提醒**：由於使用的是 OpenRouter 的**免費額度**，此管道可能受到官方的速率限制（Rate Limits，通常為 20 RPM/200 RPD），在尖峰時段回應速度可能會較慢或稍有延遲。

### C. 鏈式自動故障轉移 (API Failover Sequence)
重構對話接口，將多種 AI API 串聯成一條鏈。當使用者發問時，後端會**依序嘗試**：
$$\text{Gemini 3.5 Flash} \longrightarrow \text{Groq Llama 3} \longrightarrow \text{OpenRouter Free} \longrightarrow \text{OpenAI GPT-4o-mini}$$
若鏈條上的某個 API 出現異常（如 429、401、404 等），系統會立即捕獲異常，向前端傳送轉移提示（`*[系統提示]* {Provider} 失敗，正在自動切換備用方案...`），並**立刻調用下一個備用 AI**，無縫繼續輸出。

### D. 終極模擬模式降級 (Fallback to Mock Mode)
如果上述所有的 AI API 皆呼叫失敗（或皆未配置），後端會自動無縫降級至 **模擬演示模式 (Mock Mode)**，繼續以打字機效果輸出精心設計的機器學習教學邏輯（可引導學生回答「過擬合」、「資料洩漏」及「建模流程」等關鍵字問題），保證展示過程絕對不會呈現崩潰白畫面。

### E. 前端小測驗作答狀態防呆鎖定 (Quiz State Lock)
針對前端「小測驗」面板中，學生一旦答對後，再次點擊其他錯誤選項會使答題狀態跳回「再想一下」並錯誤變更狀態的邏輯缺陷進行了修正：
* 在 `selectAnswer` 函數中，新增了 `if (answers[algoId] === true) return;`。
* 當該演算法的測驗已正確回答（狀態為 `true`）後，任何後續的點選皆會被直接忽視，鎖定「答對了」狀態，保障學習成果的 UI 呈現不會受二次誤點干擾。

## 2. Render 冷啟動 UX 改善（frontend/pages/index.js）

### 問題背景

Render 免費方案後端閒置 15 分鐘後自動暫停。前端啟動時若後端尚未就緒，`GET /api/algorithms` 失敗，過去顯示技術性錯誤訊息「目前無法讀取資料，請確認後端 API 是否已啟動」。

| 狀態 | 畫面 |

| `loading=true`，無資料 | 全頁旋轉動畫 + 「平台啟動中，請稍候…」 |

| 後端就緒 | 自動載入完整平台，不需使用者操作 |

- 自動重試：catch 後啟動 `setInterval(10000)`，後端就緒時自動清除 timer 並渲染資料

| `POST /api/run-linear-regression` | 接收使用者提供的 (x, y) 點，回傳斜率、截距、R²、預測值 |
| `POST /api/simulate-linear-regression` | 接收 n, a, b, σ² 四個參數，生成模擬資料並回傳完整迴歸結果 |

`simulate-linear-regression` 回傳內容：
- 生成的 `x_values`, `y_values`（`x ~ Uniform(-100, 100)`，`y = ax + b + N(0, sqrt(σ²))`）
- `fitted_slope`、`fitted_intercept`、`r_squared`、`rmse`
- `true_line_y`（真實線端點）、`fitted_line_y`（迴歸線端點）
- `outlier_indices`（殘差最大的前 10 個點的索引）、`residuals`

### 3.2 迭代歷程

**第一版（滑桿輸入資料點）：** 使用者可對每個 (x, y) 點拖動滑桿，按送出計算。使用者回饋希望改為「生成模擬資料」的方式。

| 滑桿 | 範圍 | 說明 |
| :--- | :--- | :--- |

| a（真實斜率） | -50 ~ 50 | 控制直線傾斜 |
| b（真實截距） | 0 ~ 100 | y 軸截距 |
| σ²（噪聲變異數） | 0 ~ 300 | 資料散亂程度 |

公式框即時顯示 `y = ax + b + N(0, σ²)` 隨參數更新。

## 1. 問題描述

**原始行為：** 首次訪問時，若 `loading=true` 且 `algorithms=[]`，頁面直接顯示一個全頁居中的旋轉動畫（「平台啟動中，系統初始化中，請稍候…」），使用者完全看不到頁面結構，直到資料載入完畢才突然跳出完整頁面。

## 2. 解決方案

| 後端 warming-up（冷啟動） | 全頁雲朵提示（保留） | 全頁雲朵提示（保留） |
| 資料載入完成 | 頁面閃現 | 骨架平滑替換為真實內容 |

# Phase 5 互動演算法實驗室全面建置工作報告（2026-06-09）

本報告記錄第十三至十五階段的核心工作：為十大機器學習演算法各自建置完整的互動模擬實驗室，涵蓋後端模擬 API 與前端 SVG 可視化元件。

## 1. 工作概覽

| 子階段 | 演算法 | 狀態 |

## 2. 架構設計原則

所有實驗室遵循統一的設計模式：

- **一次 API 呼叫**：元件掛載時呼叫後端取得完整模擬結果（或在模型參數變更時重新呼叫）
- **全客戶端互動**：不影響模型的參數（如門檻值、顯示的主成分數）純前端計算，不觸發 API
- **`useMemo` 驅動**：所有衍生幾何（座標轉換、邊界線、統計值）以 useMemo 計算，依賴陣列精確宣告
- **統一視覺語言**：amber `#f59e0b` 作為強調色、blue `#60a5fa` 為類別 0、red `#f87171` 為類別 1

## 3. Phase 5-1：四大演算法實驗室

### 3.1 邏輯迴歸實驗室（LogisticRegressionLab.jsx，id=1）

**後端 API：** `POST /api/simulate-logistic-regression`
- 200 點二元分類資料（2 特徵），回傳各點的 `prob`（sigmoid 輸出）
- 回傳 `coef [w1, w2]` 與 `intercept`

**前端核心：**
- 門檻值滑桿（0.05–0.95）純前端計算，不重新呼叫 API
- 決策邊界公式：`x2 = (logit(threshold) - w1*x1 - b) / w2`
- 即時混淆矩陣（TP/FP/FN/TN）、Accuracy / Precision / Recall / F1
- 正確分類：實心圓；誤分類：虛線外框圓；邊界線：amber，非 0.5 時為虛線

### 3.2 KNN 實驗室（KNNLab.jsx，id=5）

**後端 API：** `POST /api/simulate-knn`
- 200 點 2D 訓練資料（random_state=7）

**前端核心：**
- 26×26 決策網格（676 格 × 200 訓練點），全客戶端 JS KNN，`useMemo` 快取
- X/Y 滑桿定位測試點（-3.0~3.0），即時顯示：決策區域背景、K 個最近鄰連線、鄰居高亮
- 統計：預測類別（帶色邊框）、第 K 近鄰距離、各類別票數

### 3.3 樸素貝葉斯實驗室（NaiveBayesLab.jsx，id=6）

**後端 API：** `POST /api/simulate-naive-bayes`
- 自動選取分離度最高的特徵（最大 |μ₁−μ₀|/平均σ）
- 回傳兩類別的 μ、σ、prior

**前端核心：**
- `buildChart(result)` 只依賴 `result`，高斯曲線渲染與 prior 滑桿完全解耦
- Prior 滑桿移動時只重算 amber 決策邊界線，曲線永不重繪
- `findBoundaries` 掃描 2000 個 x 值找對數後驗交叉點
- 兩排 rug plot 刻度標示實際資料分佈

### 3.4 K-Means 實驗室（KMeansLab.jsx，id=7）

**後端 API：** `POST /api/simulate-kmeans`
- 接收 `n_clusters`，回傳點群標籤、群心座標、Inertia、Silhouette Score

**前端核心：**
- K 滑桿（1–10）觸發重新呼叫 API
- 10 色陣列，各群心以 ✕ 符號標記
- K ≠ 4（真實群數）時顯示提示框

## 4. Phase 5-2：三大演算法實驗室

### 4.1 SVM 實驗室（SVMLab.jsx，id=4）

**後端 API：** `POST /api/simulate-svm`
- 接收 `C`（正則化參數），線性核 SVC
- 回傳 `coef [w1,w2]`、`intercept`、`margin_width`（= 2/||w||）、`support_vectors`、`accuracy`

**前端核心：**
- C 預設按鈕（0.01/0.1/0.5/1/2/5/10/50/100）搭配「套用」確認，避免多餘 API 呼叫
- `boundary` useMemo：主決策邊界線 + 上下邊距虛線（margin band 填色）
- 支援向量高亮：帶色彩外框圓圈
- 統計：準確率、邊距寬度、支援向量數

### 4.2 隨機森林實驗室（RandomForestLab.jsx，id=3）

**後端 API：** `POST /api/simulate-random-forest`
- 訓練 11 個模型（n = 1,3,5,10,20,30,50,75,100,150,200），生成準確率曲線
- 回傳 `curve`、`importances`（10 個特徵排序）、`final_acc`

**前端核心：**
- 滑桿選取曲線上的點，純前端高亮（不重新呼叫 API）
- 折線圖含面積填色，點擊圓點跳至對應 n 值
- Top-5 特徵重要性橫向條形圖

### 4.3 PCA 實驗室（PCALab.jsx，id=8）

**後端 API：** `POST /api/simulate-pca`
- 10 特徵 → PCA(n_components=10)，回傳 PC1/PC2 散點、EVR 陣列、cumulative EVR

**前端核心：**
- n_components 滑桿純前端高亮 EVR 長條圖（不重新呼叫 API）
- PC1 vs PC2 散點圖（二分類著色）
- 累積解釋變異量即時計算顯示
